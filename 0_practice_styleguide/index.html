Coding Style Guidelines

Coding Style Guidelines - Naming

Use meaningful names
  • Types (e.g. classes): mixed case starting with upper case
    class Line, class SavingsAccount
  • Variables: mixed case starting with lower case
    Line line; int numberOfLines;
  • Member variables: start with the prefix "_" or "m_", it's up to you but stick to it
    int _balance; double m_totalAmount;
  • Methods or functions: verbs and written in mixed case starting with lower case
    getName(), computeTotalWidth()

Coding Style Guidelines - Naming

  • Named constants (including enumeration values): uppercase using underscore to separate words
    MAX_ITERATIONS, COLOR_RED, PI
Code formatting
  • Use blocks (marked by "{", "}") also for a single statement
    if(x == 0){
 cout << x;
}
  • Block layout: Opening bracket on the same line
    while(x < 10){
 cout << x;
 x++;
}

Coding Style Guidelines

Source code comments

  • In general, the use of comments should be minimized by making the code self-documenting by appropriate name choices and an explicit logical structure
  • Add inline comments for your thoughts or complex algorithm
  • Use comments to simplify understanding of the code
